# CSV文件支持和短线模式优先检索功能

## 功能概述

系统现在支持从ku目录加载CSV文件（如《屠龙表 - 短线模式.csv》），并在用户询问"短线模式"相关问题时，优先检索和使用CSV文件中的内容。

## 实现的功能

### 1. CSV文件加载
- ✅ 支持读取`.csv`格式文件
- ✅ 自动处理多种编码（utf-8、gbk、gb2312、utf-8-sig）
- ✅ 将CSV表格转换为结构化文本格式
- ✅ CSV文件被标记为优先级1（最高优先级）

### 2. 智能问题识别
当用户问题包含以下关键词时，系统会自动识别为"短线模式"问题：
- 短线模式
- 短线
- 模式
- 打板
- 低吸
- 半路
- 接力
- 龙头

### 3. 优先检索策略
识别到短线模式问题后，系统会：
- ✅ 使用专门的查询扩展（包含"屠龙表短线模式"等关键查询）
- ✅ 在重排序时给CSV文件额外加分（+2分）
- ✅ 给"短线模式.csv"文件额外加分（+3分）
- ✅ 在日志中显示"⭐ 提升 屠龙表 - 短线模式.csv 权重"

### 4. 结构化回答
系统会基于CSV文件内容，按照以下结构组织回答：
```
## 短线模式体系概览
- 一波流接力模式
- 趋势接力/龙头接力主升模式
- 纯趋势模式

## 各模式详细介绍
对每个模式，包含：
- 行情种类
- 适用周期
- 胜率高场景
- 盈亏比
- 模式条件
- 买点
- 卖点
- 仓位
- 注意事项

## 实战要点
总结关键执行细节和常见误区
```

## 常见问题

### Q: 如何添加新的CSV文件？
A: 直接将CSV文件放到`ku/`目录下，系统会自动加载。

### Q: 如何确保某个CSV文件被优先使用？
A: 在文件名中包含优先级关键词（如"屠龙"、"短线"、"模式"等），系统会自动提高其优先级。

### Q: CSV文件编码问题怎么办？
A: 系统会自动尝试utf-8、gbk、gb2312、utf-8-sig四种编码，通常能正确解析。

## 注意事项

1. CSV文件应该有清晰的列结构
2. 第一行应该是列名
3. 避免空行和空列
4. 文件名应该包含有意义的关键词
5. 确保CSV文件使用常见编码（UTF-8或GBK）
